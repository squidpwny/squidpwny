CORS PoC (POST Request) for Strapi Admin Panel

This page, hosted on a different origin (e.g., `attacker.com`), attempts to make a POST request to `https://cms.bumba.global/api/auth/change-password`. If the target server has a misconfigured CORS policy (e.g., `Access-Control-Allow-Origin: *` and `Access-Control-Allow-Credentials: true`), the response will be displayed below. A 403 (Forbidden) error indicates the request *reached* the server but was denied, which is a success for CORS bypass. Otherwise, the browser will block access due to the Same-Origin Policy.

Note: For a POST request to succeed with CORS, the server must properly respond to a CORS preflight `OPTIONS` request if the request involves "non-simple" headers or content types (e.g., `application/json`).
